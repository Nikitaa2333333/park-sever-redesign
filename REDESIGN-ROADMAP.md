## Рекомендация по стеку (обоснование)

Для сайта парка-отеля решающее — **скорость загрузки и SEO** (органика по запросам про загородный отдых). Поэтому:

- **Архитектура: статика (SSG).** Без серверного рантайма → максимально быстро и дёшево в хостинге.
- **Рекомендуемый инструмент: Astro.** Генерирует статику с **нулём лишнего JS**, интерактив — «островами» (карусели, 360°-панорама, карта, виджет бронирования грузятся точечно). Это прямой апгрейд текущей модели «Pug-партиалы», но современный и быстрый.
- **Стили:** перенести Sass-компоненты, но заменить `zoom`-вёрстку на **fluid-типографику** (`clamp()`) и **CSS custom properties** (темизация, аккуратный адаптив).
- **Убрать:** jQuery, fullpage.js, Instagram-ленту, animate.css, неиспользуемый PHPMailer/phpQuery.
- **Бронирование:** на старте — текущий **виджет `znms`/bronirui-online** (быстро, без интеграции). Когда понадобится контроль над UX/шагами/оформлением — перейти на их **API**.

> Бюджетная альтернатива: оставить Pug/Sass, но почистить мусор и мигрировать сборку **Gulp → Vite**. Дешевле, но не даёт всех плюсов островной архитектуры.

## Этапы

### Этап 0 — Подготовка

- Зафиксировать текущий сайт как эталон: скриншоты всех 13 страниц (desktop + mobile), список интеграций (см. `AUDIT.md` §3).
- Согласовать с заказчиком стек (Astro vs Pug+Vite) и дизайн-направление.

### Этап 1 — Дизайн

- Макеты ключевых страниц: главная, дом, досуг, о нас, спецпредложения, контакты. Остальные — по тем же компонентам.
- Контент берётся 1:1 из `content/*.json`.

### Этап 2 — Вёрстка и перенос контента
- Поднять проект на выбранном стеке, перенести переиспользуемые секции (header, footer, contact, map, about и т.д.).

- Реализовать интерактив островами: карусели, 360°-тур, Яндекс-карта, виджет бронирования, VK-новости, Jivo-чат.

### Этап 3 — Медиа-пайплайн (критично для скорости) — ⏸ ОТЛОЖЕНО (делаем позже)
- Встроить в сборку: генерацию **WebP/AVIF** + `<picture>`/`srcset`, `loading="lazy"`.
- Сжать видео (H.264, web-битрейт, отдельные mobile/desktop; `territory.mov` → `.mp4`).
- Шрифты → **WOFF2** с subset кириллицы.
- Цель: с ~255 МБ до **~128 МБ** (см. `AUDIT.md` §5).

### Этап 4 — Отзывы и интеграции
- ✅ Все интеграции зафиксированы в `content/site.json` → `integrations` (бронирование znms, Метрика, Карты, Jivo, оплата, 3D-тур, VK).
- **Отзывы:** свой блок в нашем стиле (данные — `content/reviews.json`). Scraping Яндекса убираем.
- ⚠️ **Платёж `/pay`** — сейчас тестовый контур `alfa.rbsuat.com`. Заменить на боевой шлюз до запуска.
- VK-виджет на «Новостях» позже заменить своим (страницу оставляем).

### Этап 5 — Сборка, деплой и миграция на тот же домен — ⏸ ОТЛОЖЕНО (делаем позже)

**Сборка.** Astro `npm run build` → папка `dist/` (чистая статика: HTML/CSS/JS + оптимизированные медиа). Никакого PHP/Node на проде не требуется — обычный хостинг reg.ru подойдёт.

**Деплой.** GitHub Actions нового репозитория: `build` → заливка `dist/` по FTP на reg.ru (переносим готовый `FTP-Deploy-Action`, путь `/www/park-sever.ru/`, креды в Secrets нового репо). Механизм и домен — те же, меняется только сборщик.

**Миграция на текущий домен — пошагово и безопасно (отвечает на «удалить старый сайт? риски?»):**

> Деплой не «удаляет» сайт целиком — он **перезаписывает файлы** в папке домена новой сборкой. Чтобы это было без риска:

1. **Бэкап текущего прод-сайта.** Скачать всё содержимое `/www/park-sever.ru/` по FTP в архив + сохранить снапшот. Исходники старого сайта уже есть в git (`nikkorfed/park-sever`) — это второй уровень отката.
2. **Staging-превью.** Сначала залить новый сайт в подпапку/поддомен (напр. `new.park-sever.ru` или `/new/`), проверить вживую: бронирование открывается, карты/метрика/оплата работают, все 13 страниц и медиа на месте.
3. **Cutover.** Только после проверки — заменить корень домена новой сборкой (или переключить поддомен на корень).
4. **Откат.** Если что-то не так — восстановить из бэкапа (шаг 1) за минуты.

**Риски и как закрываем:**
| Риск | Митигация |
|------|-----------|
| Перезапись живого сайта | бэкап + staging-превью до cutover |
| Битые ссылки/просадка SEO | URL 1:1 (см. `content/seo.json`), редиректы не нужны; sitemap/robots |
| Сломается бронирование | embed зафиксирован в `site.json` → проверяем на staging |
| Тестовый платёж (`rbsuat`) | заменить на боевой шлюз до запуска (см. `site.json.integrations.payment`) |
| Нет отката | старый сайт в git + FTP-бэкап |

**Качество.** E2E (Playwright): открытие виджета бронирования, навигация, формы. Lighthouse/CWV до и после.

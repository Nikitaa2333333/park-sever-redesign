## Главное правило: источник правды — JSON

Весь контент и структура сайта живут в **`content/*.json`**. Это единственный источник правды.
- Тексты — **дословно** из старого сайта (без пересказа/сокращений). Менять контент — только в JSON.
- Дизайн **читает** JSON и сам текста не содержит.
- Ничего не теряем: закомментированный в старом сайте контент сохранён с `"disabled": true` (вернуть = снять флаг).
- НЕ класть контент в markdown-файлы (md — только для аудита/планов).

## Структура проекта

```
content/               ← ИСТОЧНИК ПРАВДЫ
  SCHEMA.md            ← описание формата JSON
  site.json            ← компания, контакты, соцсети, реквизиты, integrations (embed-коды)
  seo.json             ← URL-карта (1:1), title/description, sitemap/robots, редиректы
  reviews.json         ← свои отзывы (наполнить)
  pages/*.json         ← 13 страниц (секции в порядке вывода; ref на общие)
  sections/*.json      ← переиспускаемые секции (header, footer, contact, map, about, gift-cards, ...)
AUDIT.md               ← тех/медиа-аудит старого сайта
REDESIGN-ROADMAP.md    ← план по этапам
source-repo/           ← клон старого сайта (nikkorfed/park-sever) — «карьер»: берём медиа + сниппеты, код НЕ переносим
```

## Модель контента

- Страница: `{ id, route, seo, options, sections[] }`.
- `sections[]` — по порядку; элемент это inline-секция **или** ссылка `{ "ref": "<имя из sections/>" }`.
- Секция: `{ id, kind, anchor, disabled, blocks[], items[] }`.
- Блоки: `overtitle, heading(level), subtitle, paragraph, list, button(href,style), link, image, video, iframe, gallery, html`.
- `items[]` — повторяющиеся структуры (карточки меню, дома, животные, собаки, акции, комплектация).

## Каркас-пилот (Astro)

Минимальный Astro-проект уже поднят: читает `content/*.json` и рендерит главную (структура, без финального дизайна). Дизайн накладываем по правилам `verstka/` (см. «Следующий этап» — возможна смена стека на React/Tailwind).

- Ключевые файлы: `src/lib/content.ts` (загрузка JSON + резолв `ref`), `src/components/{Section,Block,Header,Footer}.astro`, `src/layouts/Layout.astro`, `src/pages/index.astro`, `src/styles/preview.css`.

- Disabled-блоки/секции в рендере пропускаются (но в JSON сохранены).

# Дизайн-система «Север» — единый источник правды по визуалу

> Один файл отвечает за весь визуал: токены, типографика, ритм секций, поверхности, компоненты, анимация.
> **Контент сюда не входит** — он живёт в `content/*.json` (см. корневой `CLAUDE.md`).
> Реализация — `src/styles/design-system.css`. Этот MD = спека, CSS = её воплощение. Расходятся — чини CSS под MD.
> Если в коде встречаешь паттерн, противоречащий этому файлу, — правь код, **не выдумывай новое правило**. Сомневаешься — спроси человека.

**Канон зафиксирован (2026-06-21):** берём реальную реализацию главной. Единственный акцент — **navy**, золота нет. **2 шрифта:** заголовки — **Monotype Corsiva** (заказной каллиграфический), наборный текст и весь UI — **Lora (serif)**. См. «Что изменилось» в конце.

---

## 1. Философия

Редакционно-тихий премиум (референс — Alden). **Воздух важнее декора.** Глубина создаётся не тенями и не линиями, а **контрастом поверхностей** (white ↔ cream ↔ ink) + волосяной линией там, где нужна граница карточки. Ритм — пробелами и сменой фона секций.

Каждый элемент берёт **именованный класс** из этого файла. Ad-hoc inline-стили (кроме точечных отступов) — нежелательны: если повторяется — это новый класс, и он описывается здесь.

## 3. Типографика — закрытая шкала

Заголовки — **Monotype Corsiva** (заказной каллиграфический), наборный текст — **Lora (serif)**. Каждый элемент = один класс. Inline-комбинации (шрифт + размер + bold + цвет) — нет.

### Заголовки (`--font-display`, Monotype Corsiva)
| Класс | Спека | Где |
|---|---|---|
| `.hero-title` | display, `clamp(44px,7vw,84px)`, lh 1.05 | H1 (hero) |
| `.h1` | display, `clamp(38px,6vw,64px)`, lh 1.0 | крупный заголовок |
| `.h2` | display, `clamp(30px,3.6vw,46px)`, lh 1.06 | опенер секции |
| `.h3` | display, `clamp(19px,2vw,24px)`, lh 1.2 | подзаголовок |

⚠️ Каллиграфический скрипт — **вес обычный (500), `letter-spacing:0`** (фейк-болд и отрицательный трекинг ломают связные глифы). Negative tracking — только на UI/Lora, не на заголовках.

Цвет заголовка: на светлом — `--ink`; на `.sec--navy`/`.sec--ink` — белый (переопределяется автоматически).

### Наборный текст и подписи
| Класс | Спека | Где |
|---|---|---|
| `.lead` | Lora 500, 19px, lh 1.55, max `60ch` | лид под заголовком |
| `.body` | Lora 500, 16px, lh 1.6, max `64ch` | основной абзац |
| `.eyebrow` | Lora 600, 14px, **navy**, tracking 0 | надстрочник над H2 |
| `.hl` | navy | выделенное слово **внутри** заголовка (фирменный приём) |

Размер-уровня **ниже 14px не существует.**

### Шапка секции
Группа `eyebrow + H2 + лид` оборачивается в `.shead` (центр) или `.shead--left`. Вертикальный ритм — только `gap`, не `margin` на детях.

## 4. Ритм секций

| Класс | Фон | Когда |
|---|---|---|
| `.sec` | `--canvas` (белый) | дефолт; паддинг `96px 0` (моб. `64px`) |
| `.sec--cream` | `--cream` | вторая поверхность, чередование |
| `.sec--navy` | `--navy`, текст белый | акцентная полоса (CTA, контакты) — **редко и осмысленно** |
| `.sec--ink` | `--ink`, текст светлый | редкие кинематографические full-bleed |

Белый доминирует. Несколько `.sec` подряд — можно. **Не чередовать механически** white/navy/white/navy — фон по смыслу блока. Последняя секция перед тёмным футером — `.sec` (чистый контраст) или `.sec--navy` встык; «бутерброд» белое-между-navy-и-футером не делать.

## 5. Layout-паттерны (фактические классы)

- **`.wrap`** — контейнер `max-width 1200px`, паддинг `0 32px`.
- **`.hero`** — full-bleed видео/фото, `min-height 90vh`, тёмный скрим-градиент, текст по центру. Navbar `.nav` — `position:absolute`, прозрачный над hero.
- **`.split2`** — 2 колонки `1fr/1fr`, gap 64px (моб. 1 колонка). Текст + галерея (welcome).
- **`.gallery`** — мозаика `2fr/1fr × 2 ряда`, первая ячейка на 2 ряда, радиус `--r-img`.
- **`.grid3` / `.grid4`** — сетки карточек (3/4 кол. → 2 → 1 адаптивно), gap 24px.
- **`.stats`** — ряд «голых чисел» по центру, gap 64px.
- **`.cmap`** — контакты `1fr` + карта `1.2fr`, gap 48px.

## 6. Поверхности

- **Тени — нет.** Глубина = контраст white/cream/ink + волосяная линия `--fog`.
- **Карточка `.card`** — белая, радиус `--r-card`, граница `1px --fog`. На `.sec--cream` граница убирается (читается контрастом). Ховер — лёгкий зум картинки `scale(1.03)`, не тень.
- **Радиусы:** карточки `16`, картинки/карта `24`, кнопки — капсула `100`, бейдж `30`.
- **«Голые числа»** (`.stats`/`.stat__v`/`.stat__l`): число берёт глаз размером и navy-цветом, в карточку не заворачивается.

## 8. Анимация — минимум

| Назначение | Спека |
|---|---|
| Ховер-зум картинки в карточке | `transform: scale(1.03)`, `.4s ease` |
| Ховер кнопки | `opacity .9`, `.2s` |
| Скролл страницы | `scroll-behavior: smooth` |

Запрещено: parallax, spring, морфинг, кастомные scroll-reveal `@keyframes`, stagger. (Вход страницы `opacity/​y` — допустимо добавить позже единым правилом.)

## 9. Анти-паттерны (увидел — убери)

- Любой **второй акцент** или **золото** — акцент только navy.
- **Серость/opacity на наборном тексте** — всегда сплошной `--ink`.
- **Тень** любая — глубина контрастом и `--fog`.
- **Положительная разрядка** (`tracking-wide`) — только отрицательный tracking на крупных заголовках.
- **`uppercase`** на заголовках/кнопках/eyebrow. Курсив/наклон в наборном тексте и UI — нет (заголовки исключение: они намеренно каллиграфические через `--font-display`).
- **Размер < 14px.**
- **Hex инлайн** вместо токена; `gray-*`; красный.
- **Иконка/стрелка внутри кнопки.**
- **Третья поверхность** сверх white/cream/ink.
- Белая секция, зажатая между `.sec--navy` и тёмным футером («флаг»).

## Что изменилось относительно прежнего «закона» (история решения)

Прежняя редакция (закон заказчика «Благородный Север») отличалась в двух фундаментальных пунктах — **сознательно отменены** в пользу реальной реализации:

1. **Акцент:** было 4 цвета с **GOLD** `#D0B18A` → стало **navy — единственный акцент**, золото убрано.
2. **Шрифты:** было заголовки serif / текст sans → стало **заголовки Monotype Corsiva (каллиграфия, заказной) / весь остальной текст и UI — Lora (serif)**. Промежуточная Raleway-заглушка убрана.

Также: радиусы не `0`, а `16/24px`; для границы карточки допущена волосяная линия `--fog` (прежний закон запрещал любые бордеры). Если эти решения когда-нибудь пересматриваются — менять здесь, затем синхронно в `design-system.css`.
